# 07 — MLOps & Deployment

> **Goal:** Bridge the gap between model development and production, covering the tools, practices, and infrastructure needed to deploy, monitor, and maintain ML systems reliably.

## Topics Covered

| Module | Description | Level |
|---|---|---|
| [Model Serving](./model-serving.md) | Inference APIs, batching, serving frameworks (Triton, TorchServe, BentoML) | Intermediate / Advanced |
| [Containers & Orchestration](./containers-and-orchestration.md) | Docker, Kubernetes, Helm charts for ML workloads | Intermediate |
| [Monitoring & Observability](./monitoring-and-observability.md) | Drift detection, logging, alerting, observability stacks | Intermediate / Advanced |
| [CI/CD for ML](./ci-cd-for-ml.md) | Pipeline automation, MLflow, DVC, experiment tracking, model registries | Intermediate |

## Key Concepts

- **ML Pipelines vs. CI/CD Pipelines** — understanding the difference and how they compose
- **Model Registry** — versioning, staging, and promoting models through environments
- **Feature Store** — serving consistent features online and offline
- **Observability** — monitoring data drift, concept drift, and model decay

## Prerequisites

- Basic understanding of Python, REST APIs, and shell scripting
- Familiarity with Git and fundamental DevOps concepts
