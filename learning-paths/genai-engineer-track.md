# GenAI Engineer Learning Path

> For engineers who want to **build applications with LLMs, RAG, and generative models**.

## Path Overview

Focus on prompt engineering, retrieval-augmented generation, fine-tuning, and LLM deployment patterns.

## Core Modules

| Step | Module | Key Resources | Goal |
|---|---|---|---|
| 1 | [Module 04 — NLP](../05-generative-ai/README.md) | Tokenization, embeddings, Transformers | Foundation for LLMs |
| 2 | [Module 05 — GenAI](../05-generative-ai/README.md) | LLMs, prompting, RAG, agents | Core GenAI patterns |
| 3 | [Module 07 — Model Serving](../07-mlops-and-deployment/model-serving.md) | Triton, BentoML, vLLM | Deploy LLMs in production |

## Key Skills to Develop

| Skill | Tools |
|---|---|
| **Prompt Engineering** | LangChain, prompt templates, few-shot |
| **RAG Pipelines** | ChromaDB, Pinecone, Weaviate, LlamaIndex |
| **LLM Fine-Tuning** | HuggingFace, LoRA/QLoRA, Axolotl |
| **LLM Serving** | vLLM, TGI, Ollama, Triton |
| **Evaluation** | RAGAS, LangSmith, MLflow evaluation |
| **Guardrails** | Nemo Guardrails, Guardrails AI, prompt injection defense |
